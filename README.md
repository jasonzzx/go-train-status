# GO Train Status

A mobile-first PWA for GO Transit rail commuters. Supports all 7 GO rail lines (Lakeshore West, Lakeshore East, Stouffville, Richmond Hill, Barrie, Kitchener, Milton) — pick your line and home station, the office terminus is always **Union Station**. Shows live departure schedules, real-time platform/delay status, service alerts, and expandable stop-by-stop breakdowns.

Live: https://go-train-status.vercel.app/

---

## Tech Stack

- **Next.js 14** (App Router, `'use client'` page)
- **TypeScript**
- **Tailwind CSS** with GO Transit colour palette
- **Vercel** for hosting (auto-deploys from GitHub `main`)

## Data Sources

### Line Registry (`src/lib/lines.ts`)
Per-line display metadata (name, colour, default home station) plus the GTFS-derived station list and schedule for each of the 7 GO rail lines, loaded from `src/data/schedules/*.json` (generated by `scripts/generate-schedule.mjs` from the official GO GTFS feed). Every line's station list ends at Union Station — the app treats `UN` (Union) as the fixed office terminus regardless of which line/home station the user picks.

### Schedule (`src/lib/schedule-data.ts`)
Line-agnostic trip + stop-sequence logic built on top of `lines.ts`. Given a `lineId`, returns trips for any station/direction/service type on that line. `vehicleType: 'train' | 'bus'` distinguishes train trips from express bus trips (direct, no intermediate stops).

### Live Tracker (`/api/tracker`)
Fetches both direction pages from [railsix.com](https://railsix.com) in parallel, using the selected line's home-station `railsixSlug`:
- `https://railsix.com/routes/{homeSlug}-to-union` (SB / homeToOffice)
- `https://railsix.com/routes/union-to-{homeSlug}` (NB / officeToHome)

railsix.com is a SvelteKit SSR app — trip data is embedded as:
```js
__sveltekit_XXXX.resolve(1, () => [[{ scheduledAt, actualAt, platform, status, stops, ... }]])
```
The route extracts this with regex, converts JS object-literal keys to JSON, and returns a `TrackerTrip[]` array. Set `export const dynamic = 'force-dynamic'` and `cache: 'no-store'` so Vercel never caches stale data.

### Service Alerts (`/api/alerts`)
Fetches the GO Transit service alerts RSS feed for the selected line code (`ST`, `LW`, `LE`, `BR`, `RH`, `KI`, `MI`) and parses it with `cheerio`.

## Intermediate Stops

Each line's full outward-to-Union station list (including intermediate stops) comes from `lines.ts` / `src/data/schedules/*.json`. Bus trips are direct (no intermediate stops); train trips stop at every station on the line between the selected home station and Union.

When `trackerTrips` includes a `stops: string[]` for a trip (live data from railsix), those station names override the GTFS-derived list via the `liveStops` prop on `TrainCard`.

## Auto-Update on Deploy

Since this is an installed PWA (standalone mode on iOS/Android), browsers can keep serving a stale cached build indefinitely after a new deploy. To fix that:

- `scripts/write-version.mjs` runs as a `prebuild` step and writes `public/version.json` with a `{timestamp}-{gitSha}` version string on every build.
- `next.config.mjs` sets `Cache-Control: no-cache, no-store, must-revalidate` on `/`, `/manifest.json`, and `/version.json` so the HTML shell and version probe are never cached.
- `layout.tsx` reads `public/version.json` at build time and passes it into `VersionWatcher`, a client component that polls `/version.json` every 60s (plus on tab focus/visibility change) and calls `window.location.reload()` if the server's version no longer matches the one baked into the page.

Result: any open tab or installed PWA picks up a new deployment automatically, without the user needing to force-quit and relaunch the app.

## Key Features

- **Line + home station picker** — switch between any of the 7 GO rail lines and their stations; office terminus is always Union Station
- **Direction toggle** — Home → Union / Union → Home
- **Date picker** — browse any date's schedule
- **"Next" badge** — highlights the next departure
- **"Now" badge + On Board** — amber pulsing badge and expandable progress view for in-progress trains
- **Expandable stop list** — shows intermediate stations with estimated times and a live position indicator
- **Platform + delay badges** — pulled from railsix.com every 30 seconds
- **Service alerts sheet** — bottom sheet with GO Transit alerts
- **Add to Home Screen banner** — detects iOS Safari vs Android Chrome and shows install prompt
- **PWA** — installable, standalone display mode, custom icon
